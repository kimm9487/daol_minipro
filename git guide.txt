가장 기본 & 추천 순서 (전체 변경사항 푸시)
Git Bash (또는 VS Code 터미널)에서 아래 순서대로 입력하세요.
Bash# 1. 현재 상태 확인 (꼭 먼저 해보기)
git status

# 2. 현재 브랜치 다시 확인 (안전장치)
git branch
# → * jaehun 이 표시되어 있어야 함

# 3. 모든 변경사항 추가 (전체 푸시할 때)
git add .

# 또는 이미 일부만 추가하고 싶을 때 건너뛰어도 됨

# 4. 커밋 (메시지는 본인이 원하는 대로)
git commit -m "UserList 하이라이트 기능 구현 및 스타일 수정"

# 5. 푸시
git push origin jaehun

# 처음 푸시하는 브랜치라면 아래 명령어로 한 번만 실행
# (그 다음부터는 그냥 git push 만 써도 됨)
# git push -u origin jaehun

특정 파일만 커밋 & 푸시할 때 순서
예시: UserList.jsx와 UserList.css만 올리고 싶을 때
Bash# 1. 상태 확인
git status

# 2. 원하는 파일만 추가 (staging)
git add src/pages/UserList.jsx
git add src/pages/UserList/UserList.css

# 또는 패턴으로 여러 파일 한 번에
# git add src/pages/UserList.*

# 3. 추가된 파일 확인 (녹색으로 나와야 함)
git status

# 4. 커밋 (메시지 필수!)
git commit -m "UserList.jsx & CSS - 검색어 하이라이트 기능 추가"

# 5. 푸시
git push origin jaehun


지금 바로 실행해 볼 수 있는 최소 순서 (복붙 추천)
Bashgit status                  # 변경사항 있는지 확인
git add .                   # 전부 추가
git commit -m "feat: UserList 텍스트 하이라이트 구현"
git push origin jaehun      # 푸시
